
DTO per ultimi aggiornamenti
Il DTO di ogni elemento della lista deve avere questa struttura:

JSON
{
  "eventId": "string",
  "eventType": "string", // Valori accettati: "CHAT", "EXPENSE", "CHORE"
  "title": "string", // Vedi sotto come valorizzarlo
  "subtitle": "string", // Vedi sotto come valorizzarlo
  "timestamp": "string (ISO 8601)",
  "imageBytes": "base64 string o null" // Può essere null Lo uso solo per la CHAT se c'è la foto profilo, altrimenti per le altre metti null.
  "userCode":"String"
}

Come devi valorizzare title e subtitle in base all'eventType:

Se eventType == "CHAT":
- title: Nome e cognome della persona.
- subtitle: La preview dell'ultimo messaggio.

Se eventType == "PORTFOLIO":
- title: "Nuova spesa da {nomepersona}".
- subtitle: Il nome/descrizione della spesa.

Se eventType == "CHORE":
- title: "Nuova faccenda per {nomepersona}".
- subtitle: "Pulizia {nomechore} entro {datascadenza}".

Le icone le gestisco io lato frontend in base all'eventType.
 
--------
API 

- GET /api/notifications/{userCode}

Cosa fa: Ritorna la lista completa di tutte le notifiche attive per l'utente, mischiando chat, spese e faccende.

Ordinamento: Deve essere rigorosamente in ordine cronologico decrescente (dalla più recente alla più vecchia) basandosi sul timestamp.

- DELETE /api/notifications/{eventId}

Cosa fa: Elimina una singola notifica dal DB.

Quando si usa: Quando l'utente elimina manualmente la notifica dalla UI (es. con uno swipe o cliccando su una "X").

- DELETE /api/notifications/clear/{userCode}?type={eventType}

Cosa fa: Elimina in blocco tutte le notifiche di una specifica categoria per quell'utente.

Quando si usa: Quando l'utente entra nella pagina dedicata.

-------
Logica di Creazione
Le notifiche devono essere salvate a database quando:

CHAT: Viene creata una notifica per ogni singolo nuovo messaggio ricevuto dall'utente in una delle sue chat attive.

PORTFOLIO: Viene creata una notifica solo se la nuova spesa registrata nella casa riguarda direttamente l'utente (es. ha un debito, o è una spesa condivisa a cui deve partecipare).

CHORE: Viene creata una notifica in ogni caso a chiunque appartiene la faccenda

AGGIUNTA 2 fun 
- GET /api/chores/next/{userCode}
Il backend deve scansionare le faccende della casa e restituirmi la prossima faccenda in ordine temporale assegnata a questo specifico utente.
Il DTO di risposta deve essere così:

JSON
{
  "choreCode": "string", 
  "choreName": "string", 
  "deadline": "string (ISO 8601)" 
}
(Nota: se l'utente non ha faccende assegnate, fammi tornare un 204 No Content o un DTO con choreName nullo, così la grafica mostra "Nessuna").

- GET /api/debts/total/{userCode}
Il backend deve sommare tutti i debiti che l'utente ha attualmente con tutti i suoi coinquilini, e restituirmi una singola cifra netta.
Il DTO di risposta può essere semplicissimo:

JSON
{
  "totalDebt": "Double" 
}
